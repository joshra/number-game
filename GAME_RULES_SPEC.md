# Number Gate Rush 規格文件

## 目的
本文件定義 `Number Gate Rush` 的核心玩法、數值生成、畫面穩定性與驗收條件。

這份規格的角色不是靈感筆記，而是後續開發、重構、測試時的唯一依據。若實作與本文件衝突，以本文件為準，除非文件本身被更新。

## 產品定位
- 類型：雙跑道、自動射擊、數字方塊收益遊戲
- 核心樂趣：在有限時間內判斷哪個方塊更值得打爆
- 次核心：控道、時機、風險取捨
- 非核心：精準瞄準、複雜敵人 AI、自由移動

## 核心玩法
### 關卡單位
- 一個「回合」等於一排。
- 每排最多 2 個方塊，分布在左、右兩道。
- 方塊從畫面上方向下移動。
- 畫面同時只存在：
  - 當前排
  - 下一排預覽
- 只有當前排可互動。

### 玩家規則
- 玩家只能在左道或右道。
- 角色會自動朝當前跑道正前方連續射擊。
- 首版不提供手動開火。
- 切道沒有冷卻。
- 切道本身不提供無敵、穿透、加成或減傷。

### 方塊資料結構
每個方塊都必須具備以下欄位：
- `lane`：左或右
- `hp`：當前血量
- `maxHp`：初始血量
- `rewardType`：`add` 或 `mul`
- `rewardValue`
- `state`：`falling | broken | missed`

### 方塊顯示規則
- 大數字：`hp`
- 小數字或符號：擊破後的獎勵內容，例如 `+9`、`×2`
- `add` 與 `mul` 必須有明確顏色區分

### 命中規則
- 子彈每次命中固定扣血。
- 每顆子彈只可命中一次。
- 子彈不可穿透。
- 子彈只對當前排有效。
- 過關瞬間必須清空舊子彈。
- 命中後必須立即更新方塊血量顯示。
- 當 `hp <= 0` 時，方塊狀態改為 `broken`，並立即發放獎勵。

### 獎勵規則
首版只保留正向獎勵：
- `+N`
- `×M`

首版不加入負向獎勵方塊，例如：
- `-N`
- `/M`

原因：先穩定建立「射擊掉數字到 0 再拿收益」的核心手感，再決定是否加入紅色陷阱。

### 獎勵結算順序
方塊擊破後，結算順序固定如下：
1. 方塊破壞
2. 套用方塊獎勵
3. 若有連擊獎勵，追加連擊獎勵
4. 結束本排
5. 進入下一排

### 過關規則
- 單方塊排：打爆該方塊才算成功。
- 雙方塊排：玩家只需打爆其中一個方塊即可過排。
- 一旦本排已有 1 個方塊被打爆，本排立刻結束，另一個方塊作廢。

原因：保留「左右二選一」的策略核心。若要求兩個都打爆，玩法會退化成純 DPS 檢查。

### 漏接規則
- 若本排沒有任何方塊被打爆，且當前排所有可得獎勵方塊都越過玩家判定線，視為漏接。
- 漏接後：
  - 扣固定懲罰 `-missPenalty`
  - 連擊歸零
  - 本排結束
  - 進入下一排

### 子彈規則
- 子彈只對當前排生效。
- 當前排之外的方塊不可被子彈提前打到。
- 過關後所有舊子彈必須立即清空，避免跨排殘留。

## Boss 規則
### 設計原則
- Boss 必須沿用與普通方塊同一套「掉血到 0 才擊破」規則。
- Boss 不可再使用舊版「到線直接固定扣值即結算」邏輯。

### Boss 行為
- Boss 為單一置中目標。
- 玩家無論站在左道或右道都能打中 Boss。
- Boss 必須完整進入畫面後才可受擊。
- Boss 必須被打到 `0` 才算通關。
- 若 Boss 到達玩家判定線前仍未被擊破，直接失敗。
- Boss 擊破後套用固定終局獎勵，再進入勝利結算。

### Boss 資料
Boss 至少具備：
- `hp`
- `maxHp`
- `rewardType`
- `rewardValue`
- `state`

## 連擊規則
- 成功打爆一個方塊並完成本排時，`combo +1`
- 以下情況 `combo = 0`
  - 漏接
  - Boss 撞線失敗
  - 主動重開
- 切道不影響連擊
- Boss 戰不累積一般連擊
- 每 `3` 連給固定獎勵
- 連擊獎勵建議只使用 `+N`，不用乘法，以控制波動

## 數值生成規則
### 掉落模式分布
每排模式建議：
- 左單方塊：30%
- 右單方塊：30%
- 雙方塊：40%

### 雙方塊排設計原則
雙方塊排中，兩邊必須有明顯差異：
- 一邊低血低報酬
- 一邊高血高報酬

不允許兩邊只是純亂數且沒有策略差異。

### 平衡原則
以下規則必須可驗證：
- 第 1 到 3 排不得出現雙邊都需要超過 1 秒持續射擊才可擊破的情況
- 第 1 到 3 排至少要有 1 邊是新手可穩定打爆的低血方塊
- `mul` 應低頻高價值
- `add` 應高頻穩定
- 高血方塊必須對應明顯更高的收益
- 不允許生成「高血量但低收益」的陷阱方塊，除非未來正式加入負向設計
- 不允許前 3 排出現必死配置

## 畫面穩定性規格
### 鏡頭規則
- 鏡頭只能單方向、等速移動。
- 不允許：
  - 過關時瞬間重設 `cameraOffset`
  - 根據當前排重新定位鏡頭
  - 因命中、破壞、過關造成整體畫面上下跳動

### 排間距規則
- 每排邏輯間距固定，例如 `rowGap = 132`
- 隨機性不能改變排的邏輯位置，只能影響排內視覺表現
- 排與排的生成時間固定
- 排與排的碰撞順序固定
- 不允許因隨機 offset 導致某排超車、貼近前一排或跳位

### 隨機偏移規則
- 垂直偏移 `offsetY` 只供視覺用途
- `offsetY` 範圍限制：`-12px ~ +12px`
- 水平偏移：`0`
- 不允許旋轉
- 不允許縮放跳動
- 偏移不能影響命中框、互動時機或漏接判定

### 命中框穩定性
- 命中判定必須綁定邏輯框，不綁特效
- 命中時可做：
  - 閃白
  - 短暫縮放 `0.96 ~ 1.0`
  - 數字跳動
- 命中時不可：
  - 改變世界座標
  - 改變碰撞框位置
  - 改變整排下落速度

### 破壞動畫穩定性
- 方塊破壞後，邏輯上必須立刻停止碰撞
- 視覺上可播放 `150ms ~ 250ms` 的獨立破裂特效
- 不允許因動畫延遲而讓同一方塊再吃第二次子彈

### 子彈穩定性
- 子彈速度固定
- 不允許因命中、過關、Boss 出場、連擊獎勵而改變既有子彈速度或方向
- 若規則要求過關清子彈，必須直接清空，不做會誤導的反彈或吸走效果

### HUD 穩定性
- HUD 數字更新不能改變版面高度
- HUD 項目需要固定寬度或最小寬度
- 數字位數增加不得推擠其他欄位
- 進度、連擊、隊伍值更新時不得產生 layout shift

### 字體與數字穩定性
- 方塊上的大數字與小獎勵文字必須中心對齊
- 數字位數變化不應導致左右跳動
- 若數字過大，應縮字，不應溢出或推動方塊尺寸

### 視覺層級
渲染層級固定如下：
1. 背景
2. 軌道
3. 當前排與下一排方塊
4. 子彈
5. 玩家
6. HUD / 浮字 / Overlay

### 可視範圍規則
- 只有方塊外框完全進入可視區後才開始互動
- 離開畫面後立即停止渲染與互動
- 「完全進入」以邏輯矩形計算，不看陰影、發光、粒子

### 每幀更新順序
每一幀的處理順序必須固定：
1. 更新邏輯位置
2. 更新子彈位置
3. 執行碰撞判定
4. 更新物件狀態
5. 最後渲染

不允許先渲染舊位置，再用新位置碰撞。

### 過關切換穩定性
- 一排結束時只更新 `rowIndex`
- 清空舊子彈
- 下一排依既有速度自然接續
- 不允許：
  - 畫面瞬間拉回
  - 下一排瞬移
  - 下一排突然加速

### Boss 出場穩定性
- Boss 必須沿用與普通排相同的世界座標移動規則
- 不允許：
  - 瞬移到畫面中央
  - 先隱形後突然完整出現
  - 進場時推動整個鏡頭

## 驗收條件
### 邏輯驗收
以下必須全部成立：
- 畫面外方塊不可互動
- 過關不殘留舊子彈
- 單方塊排可正常成功與漏接
- 雙方塊排打爆任一方塊即過排
- Boss 必須打到 `0` 才通關
- 不存在無獎勵可打的空排
- 不存在前 3 排必死配置

### 畫面穩定驗收
以下任一發生，視為不符合規格：
- 過關時整體畫面上下跳動
- 下一排出現高度瞬移
- 方塊命中後位置改變超過 `2px`
- 同一物件在未互動時出現抖動
- HUD 因數字更新而推擠版面
- Boss 出場不是平滑進入，而是突然出現

## 目前鎖定的關鍵決策
- 首版只保留正向獎勵，不做負向方塊
- 雙方塊排打爆任一個就過關
- 當前排之外不可互動
- Boss 完全沿用方塊掉血規則
- 高度隨機只做小幅視覺變化，不改變排邏輯
